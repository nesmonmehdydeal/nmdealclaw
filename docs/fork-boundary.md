# nmdealclaw fork boundary

## Reuse doctrine

`nmdealclaw` reuses ideas and selected code carefully, while remaining a distinct project.

## Useful influences

### ClawWork
- task-first workspace logic
- artifact-first UX
- durable task persistence
- explicit run control

### HiClaw
- visible manager-worker orchestration
- human-in-the-loop intervention points
- shared-work-product mindset

### NullClaw
- lightweight runtime posture
- separate UI/runtime boundary
- adapter-friendly backend model

## Project boundary

`nmdealclaw` should keep a neutral internal domain model and avoid collapsing all upstream projects into one undifferentiated codebase.
